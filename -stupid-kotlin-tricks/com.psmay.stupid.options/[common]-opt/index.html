Opt

@JvmInline
value class Opt<out T>
Content copied to clipboard

Type to unambiguously represent either

  • the value of an actual result, or

  • the absence of an actual result.

In ordinary applications, this distinction is implemented using null; this type works in situations where this is not possible; in particular, it addresses the possibility that null is a valid value for a given purpose and thus cannot represent the absence of a value.

An Opt can be created using methods on the companion object, particularly Opt.valueOf(). More importantly, methods are available for fetching single items (firstOpt, lastOpt, elementAtOpt, singleOpt) from iterables and * sequences. Virtually anywhere a *OrNull method exists, there should be a *Opt method to report the same result as an Opt.
